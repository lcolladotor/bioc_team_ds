17.1 R setup

  • Positron IDE from https://positron.posit.co/
    • Previously we used RStudio Desktop Preview version: https://rstudio.com/products/rstudio/download/preview/
  • R: latest versions that match Bioconductor release and devel
    • macOS: you can download the latest ‚Äúbranch‚Äù and ‚Äúdevel‚Äù from https://mac.r-project.org/
    • Windows: https://cran.r-project.org/, also needs Rtools42 https://cran.r-project.org/bin/windows/Rtools/
  • RSwitch
    • macOS: https://rud.is/rswitch/
    • Windows: I install the R versions in two different folders and switch them using RStudio‚Äôs global configuration options https://cran.r-project.org/

  • Air for auto-code formatting in RStudio and other editors: https://posit-dev.github.io/air/editor-rstudio.html
    • See also https://lcolladotor.github.io/jhustatcomputing/posts/01-welcome/#how-to-get-air-auto-code-formatter

As I described in my blog post on updating R, I also like to saved my list of currently installed packages.

## Locate laptop path for where I save the list
## of installed packages
installed_path <- ifelse(
    .Platform$OS.type != "windows",
    "~/Dropbox/Computing/R",
    "C:/Users/fellg/Dropbox/Computing/R"
)
## At JHPCE I use:
# installed_path <- "~/R/update_R"

## Locate installed R packages
installed <- dir(.libPaths())
## For JHPCE:
# installed <- dir(.libPaths()[1])

## Save list of installed packages for later use
save(installed, file = file.path(installed_path, paste0(Sys.Date(), '-installed.Rdata')))
## Load previous "installed" packages
previous <- dir(path = installed_path, pattern = 'installed.Rdata')
load(file.path(installed_path, previous[length(previous)]), verbose = TRUE)

## Locate current packages
current <- dir(.libPaths())
## Use the following at JHPCE:
# current <- dir(.libPaths()[1])

## Missing ones
installed[!installed %in% current]

## Install missing packages
BiocManager::install(installed[!installed %in% current])

## List new R packages
current[!current %in% installed]

17.2.4 ~/.air.toml

As noted in the 2025-03-14 Automatic R/Bioconductor code formatting LIBD RStats club session (check the notes here), I recommend:

  • Use styler for auto-styling many files without having to open them one by one
    • biocthis dev/04_update.R example.
    • spatialDLPFC code/update_style.R example.
    • Configure your ~/.Rprofile for styler to use biocthis::bioc_style() as shown here.
  • Use Air for styling code within Positron and RStudio (or your favorite editor supported by Air)
    • When needed use # fmt: skip file or # fmt: skip due to this issue.
    • Use a global (user level) ~/air.toml file instead of one per project as shown below.
  • Configure Positron or RStudio to style files on save.
    • For Positron check https://lcolladotor.github.io/jhustatcomputing/posts/01-welcome/#how-to-get-air-auto-code-formatter.

[format]
line-width = 80

Useful for altering the behavior of the up and down arrow keys. See the companion lines on the ~/.bashrc file.

"\e[B": history-search-forward
"\e[A": history-search-backward
